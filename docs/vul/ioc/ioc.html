VUL IoC¶

  • Usage

  • API

  • Scope API

IoC principle is realized in this library as an IoC Container.

  • Is non-intrusive (does not use preprocessor directives / special types).

  • Exception-less (can be compiled with -fno-exceptions).

  • Lightweight (around 12 bytes for each object apart from the object itself).

  • Supports both transient and singleton lifetime.

  • Has built-in memory management.

  • Uses C++20 features.

Usage¶

Let’s assume we have the classes, listed below.

class Repo
{
public: virtual int get() = 0;
};

class const_repo : public Repo
{
public: virtual int get() override { return 5; }
};

class Logger
{
public: virtual void log() = 0;
};

class cout_logger : public Logger
{
    Repo* _repo;
public:
    cout_logger(Repo* repo) : _repo(repo) {}
    virtual void log() override { std::cout << _repo->get(); }
};

This is a standard Repository-Service pattern, a piece of the Onion Architecture. A repository accesses data (in our case a const value) and a service uses a repository to process the accessed data. In our case a logger class logs an int from the repo class. Now the IoC Container usage.

void example()
{
    di d;
    if (d.add_transient<Repo, const_repo>() == 0) return;
    if (d.add_singleton<Logger, cout_logger, Repo>() == 0) return;
    d.init();
    auto logger = d.get<Logger>()->get();
    logger->log(); //We get "5" in console
    d.term();
}

Danger

Do not call init() or term() more than once for one instance of di. This leads to undefined behavior.

Important

This class does not terminate on destruction, you have to call term() manually.

Note

This class does not use any special thread safety mechanisms.

Hint

It is safe to call di::free(ptr) in a destructor, if you are sure that the pointed value isn’t used anywhere else. In case of an expected error you’ll get a 0 as a return.

template<class _T, class ...Args>

Construct a class using the values, provided by the IoC Container. The value is not tracked by the memory manager.

Return

The constructed object if succeeded, nullptr otherwise.

Template Parameters

  • _T: The type of object to construct.

  • Args: The arguments, that will be passed to the constructor of _T, no pointer or reference needed. All of the arguments inside the constructor have to be pointers.

template<class _T>
void destruct(_T *value)¶

Destroy an object, constructed by the IoC Container.

Does not destroy or free the scopes, that were passed as constructor parameters. It is upon the user to check if the object was constructed by the IoC Container instance.

Template Parameters

  • _T: The type of object to destroy.

Parameters

  • [in] value: The object to destroy.

allocator_type<std::byte> &get_allocator()¶

Get the internal allocator, used by the IoC Container.

template<class _InT, class _ImV, class ...Args>
int add_transient()¶

Add an interface and implementation to the transient mapping cache.

Add a transient factory method to create objects of type _ImV, when requested using the interface of type _InT. On every subsequent call of get<_InT>() a new instance of _ImV will be constructed.

Template Parameters

  • _InT: The interface to use for mapping. Use this type to get instances in the future.

  • _ImV: The implementation to use for mapping. Objects of this type are constructed, when calling get<_InT>().

  • Args: The arguments of _ImV to use during construction. All arguments are constructed by the IoC Container, so it is not possible to pass additional arguments.

template<class _InT, class _ImV, class ...Args>
int add_singleton()¶

Add an interface and implementation to the singleton mapping cache.

Add a singleton factory method to create objects of type _ImV, when requested using the interface of type _InT. On every subsequent call of get<_InT>() the same instance of _ImV will be returned.

Template Parameters

  • _InT: The interface to use for mapping. Use this type to get instances in the future.

  • _ImV: The implementation to use for mapping. Objects of this type are constructed, when calling get<_InT>().

  • Args: The arguments of _ImV to use during construction. All arguments are constructed by the IoC Container, so it is not possible to pass additional arguments.

class vul::ioc::di_basic_scope¶

A basic scope class, holding the value, generated by the IoC Container. It shouldn’t be constructed by itself. Use the inherited classes.

Subclassed by vul::ioc::di_scope< _Ty >

Public Functions

di_basic_scope(bool *is_scope, void *value, void *di)¶

Construct a basic scope with a value and additional parameters, that indicate state.

Parameters

  • [in] is_scope: Indicates if the current scope object is active.

  • [in] value: The value, constructed by the IoC Container.

  • [in] di: A pointer to the IoC Container instance, used for cross-referencing.

~di_basic_scope()¶

A virtual destructor to ensure correct destruction of descendants.

int put(void *value)¶

Put a new value inside the scope. Disregards the old value and breaks automatic memory management.

The value, cleared by the automatic memory manager, will be the original, not the replacement. In order to use this function, the user has to manually deallocate the replacement value.

Return

1 if the replacement succeeded, 0 otherwise.

Parameters

  • [in] value: The replacement value for the scope object.

int clear()¶

Clear the value inside the scope. Sets the stored value to nullptr. Disregards the old value and breaks automatic memory management**.

The value, cleared by the automatic memory manager, will be the old value, disregarding this clear procedure. In order to use this function, the user has to manually deallocate the replacement value.

Return

1 if the replacement succeeded, 0 otherwise.
